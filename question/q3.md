whatapagetn cr 이 아래와 같이 정의됐다.
변경전에서 변경후 로 바뀌면 자동으로 whatap-node-agent 데몬셋이 삭제되는게 정상적인 오퍼레이터 동작방식인가?
내가 실제로 테스트 했을때는 저렇게 해도 노드에이전트가 삭제되지는 않는거같다.

```yaml (변경전)
apiVersion: "monitoring.whatap.com/v2alpha1"
kind: "WhatapAgent"
metadata:
  labels:
    app.kubernetes.io/managed-by: "whatap-operator"
    app.kubernetes.io/name: "whatap"
  name: "whatap"
  namespace: "whatap-monitoring"
spec:
  features:
    k8sAgent:
      customImageFullName: "public.ecr.aws/whatap/kube_mon:1.8.9"
      gpuMonitoring:
        customImageFullName: "public.ecr.aws/whatap/dcgm-exporter:4.4.2-4.7.09-ubuntu22.04"
        enabled: true
      masterAgent:
        enabled: true
      nodeAgent:
        enabled: true
        hostNetwork: true
        runtimeClassName: nvidia
        hostPID: true
        runtime: "containerd"
        runtimeSocketPath: "/var/run/containerd/containerd.sock"
```
